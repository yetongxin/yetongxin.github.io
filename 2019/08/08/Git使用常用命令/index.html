版本回退

HEAD指向的版本就是当前版本，因此，Git允许我们在版本的历史之间穿梭，使用命令git reset –hard commit_id
穿梭前，用git log可以查看提交历史，以便确定要回退到哪个版本。
要重返未来，用git reflog查看命令历史，以便确定要回到未来的哪个版本。

Git 撤销修改

checkout -- readme.txt
改乱了工作区的内容（未add），丢弃工作区可以用
改乱了工作区的内容（已经add），丢弃工作区可以用：git reset HEAD <file>后checkout -- readme.txt

Git 删除

git rm删掉，并且git commit

Git 恢复删除

删错时候因为版本库里还有，所以可以很轻松地把误删的文件恢复到最新版本：
$ git checkout – filename
git checkout其实是用版本库里的版本替换工作区的版本，无论工作区是修改还是删除，都可以“一键还原”。

关联远程库

要关联一个远程库，使用命令
git remote add origin git@server-name:path/repo-name.git；
关联后，使用命令git push -u origin master第一次推送master分支的所有内容；
此后使用命令git push origin master推送最新修改；

分支

  • 创建分支：git checkout -b dev 创建分支并切换

  • 切换分支：git checkout <name>

  • 列出所有分支：git branch命令会列出所有本地分支，git branch -a列出所有远程与本地分支

  • 合并分支：git merge命令用于合并指定分支到当前分支

  • 删除分支：git branch -d dev

  • 把某个分支上的内容都拉取到本地 git pull origin dev

  • 拉取远程分支并创建本地分支：git checkout -b 本地分支名x origin/远程分支名x，此种方法建立的本地分支会和远程分支建立映射关系
    第二种方法是：git fetch origin 远程分支名x:本地分支名x，然后手动Checkout，此种方法建立的本地分支不会和远程分支建立映射关系。

    • 本地分支和远程分支建立映射关系的作用
      使用git pull或者git push时就不必每次都要指定从远程的哪个分支拉取合并和推送到远程的哪个分支了。
      git branch -vv查看映射关系

  • 手动建立本地分支和远程分支的映射关系：git branch -u origin/分支名(没用过)，或者git branch --set-upstream-to origin/分支名(用过)

  • 撤销本地分支与远程分支的映射关系：git branch --unset-upstream

  • 是否和远端的仓库建立连接：git remote -v

    实际开发的分支策略

    在工作的时候我们应该在dev分支上干活，老大负责把dev分支merge到master

在git clone下来之后，git checkout -b develop（本地分支） origin/develop（或者最新的版本分支master），此种方法建立的本地分支会和远程分支建立映射关系。
接下来基于develop分支下创建一个feat/featurename，
然后推送push到远程，因为现在远程还没有feat/featurename分支，所以需要set-upstream,第二次及以后就可以直接Push了：
git push --set-upstream origin feat/featurename

后续Push可以先pull一下，然后手动解决冲突后add commit push

取远程分支的方法：

  1. git init
    git remote add origin git@github.com:XXXX/nothing2.git //与origin master建立连接
    git fetch origin dev（dev为远程仓库的分支名）//把远程分支拉到本地（这里开始下载）
    git checkout -b dev(本地分支名称) origin/dev(远程分支名称) //在本地创建分支dev并切换到该分支
    git pull origin dev(远程分支名称) 把某个分支上的内容都拉取到本地//把某个分支上的内容都拉取到本地
  2. git clone
    git checkout -b develop（本地分支） origin/develop 选择一个远程分支

git fetch和git pull区别：

git fetch <远程主机名> //这个命令将某个远程主机的更新全部取回本地,只是手动指定了要fetch的remote。在不指定分支时通常默认为master
git fetch <远程主机名> <分支名> //只想取回特定分支的更新,如取回master: git fetch origin master
git fetch是将远程主机的最新内容拉到本地，用户在检查了以后决定是否合并到工作本机分支中。
而git pull 则是将远程主机的最新内容拉下来后直接合并，即：git pull = git fetch + git merge，这样可能会产生冲突，需要手动解决。

git pull 的过程可以理解为：
git fetch origin master //从远程主机的master分支拉取最新内容
git merge FETCH_HEAD //将拉取下来的最新内容合并到当前所在的分支中
